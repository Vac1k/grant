# Як ми збираємо інформацію про гранти

Цей документ простими словами пояснює, як інструмент отримує інформацію про гранти з сайтів.

## Коротко

Ми не завантажуємо весь сайт повністю.

Замість цього інструмент знаходить сторінки, де перелічені грантові можливості, відкриває обмежену кількість сторінок конкретних грантів, зберігає оригінальну інформацію і потім витягує з неї корисні поля.

Наприклад, якщо ліміт встановлено на `20`, інструмент намагається зібрати до 20 грантових можливостей з кожного налаштованого джерела за один запуск.

## Що відбувається, якщо сайт має API

Деякі сайти дають структуровані дані через API.

API можна уявити як офіційний вхід до даних. Інструмент не читає видиму сторінку сайту, а запитує у сайту дані у зручному структурованому форматі.

Це найкращий варіант, бо інформація зазвичай чистіша, стабільніша і легше обробляється.

Приклади:

- EU Funding & Tenders Portal;
- Diia Business finance programs, де доступний публічний frontend API.

Для таких джерел інструмент запитує записи через API, а потім зберігає потрібну інформацію в базу даних.

## Що відбувається, якщо сайт не має API

Багато грантових сайтів не мають зручного API.

Для таких сайтів інструмент працює обережно й поетапно:

1. Відкриває сторінку зі списком грантів.
2. Знаходить посилання, які схожі на грантові можливості.
3. Бере тільки обмежену кількість посилань, наприклад 20.
4. Відкриває кожну окрему сторінку гранту.
5. Зберігає оригінальний HTML або текст як raw snapshot.
6. Витягує корисні поля зі сторінки.
7. Зберігає очищений запис гранту в базу даних.

Процес виглядає так:

```text
сторінка зі списком грантів -> посилання на гранти -> окремі сторінки грантів -> збережені записи
```

Після досягнення ліміту інструмент не продовжує обходити весь сайт.

## Що означає ліміт

Якщо запустити:

```bash
grant-tool ingest --source gurt --limit 20
```

інструмент спробує зібрати до 20 грантових можливостей з GURT.

Якщо запустити:

```bash
grant-tool ingest --all --limit 20
```

інструмент спробує зібрати до 20 можливостей з кожного налаштованого джерела.

Для поточних MVP-джерел це означає:

- до 20 з EU Funding;
- до 20 з Prostir;
- до 20 з Diia Business;
- до 20 з GURT.

Тобто за один запуск може бути зібрано приблизно до 80 можливостей, якщо кожне джерело має достатньо доступних записів.

## Чому ми не завантажуємо все підряд

Завантажувати весь сайт зазвичай погана ідея.

Причини:

- багато сторінок не мають жодного стосунку до грантів;
- це може працювати повільно;
- це може зайво навантажувати публічні сайти;
- це створює багато непотрібних даних;
- різні сайти мають різну структуру;
- частина сторінок може бути старою, дубльованою або нерелевантною.

Тому інструмент фокусується тільки на сторінках, які з високою ймовірністю містять грантову інформацію.

## Яку інформацію ми зберігаємо

Для кожної зібраної можливості інструмент зберігає два типи даних.

Перший тип - оригінальні raw дані:

- оригінальна HTML-сторінка;
- оригінальна відповідь API, якщо вона є;
- очищений текст сторінки;
- посилання на джерело;
- статус запиту і технічні метадані.

Це потрібно, щоб потім можна було перевірити, що саме було написано на сайті.

Другий тип - очищений запис гранту:

- назва;
- посилання на джерело;
- статус;
- дедлайн;
- сума фінансування;
- донор або організація;
- умови участі;
- типи заявників;
- теми;
- географія;
- документи;
- позначки для ручної перевірки.

Не кожен сайт чітко дає всі ці поля. Якщо інструмент не впевнений у якості витягнутої інформації, він позначає грант для ручної перевірки.

## Що відбувається після збору

Після збору грантів інструмент може:

1. очистити й нормалізувати поля;
2. порівняти гранти з профілями клієнтів;
3. оцінити релевантність кожного гранту;
4. згенерувати пояснення для найкращих збігів;
5. показати все в dashboard.

Збір грантів - це тільки перша частина всього процесу.

## Простий приклад

Уявімо, що на сайті є сторінка "Гранти".

Інструмент відкриває цю сторінку і знаходить 100 посилань на грантові можливості.

Якщо наш ліміт дорівнює 20, інструмент не відкриває всі 100.

Він відкриває перші 20 сторінок грантів, зберігає їхній вміст, витягує корисну інформацію і зупиняється.

Пізніше ми можемо запустити інструмент з більшим лімітом або покращити connector, щоб він обробляв більше сторінок.

## Важливий висновок

Інструмент не є загальним crawler-ом для всього інтернету.

Це контрольована система збору грантів.

Для кожного джерела потрібен окремий connector, який знає, де шукати гранти і як розпізнавати сторінки грантів саме на цьому сайті.
